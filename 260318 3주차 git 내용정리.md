# Git 정리

## Git이란?
- **DVCS** (분산 버전 관리 시스템)
- 소스 코드 변경 이력을 추적하고 관리
- 과거로 되돌리기, 기능 테스트, 협업 가능

---

## 사용 방법
- **CLI** : 터미널, Git Bash, cmd
- **GUI** : GitHub Desktop, SourceTree

---

## Git 설치
1. https://git-scm.com/downloads 에서 다운로드
2. 설치 시 **Git Bash** 포함 여부 확인
3. 에디터 선택 (익숙한 걸로)
4. 기본 브랜치 이름 : **main** (또는 master)
5. 개인 PC라면 **Git Credential Manager** 체크

### 설치 확인
```bash
git --version
git config --global core.autocrlf true  # 윈도우/맥 줄바꿈 오류 방지
```

## 기본 사용법

### 처음 시작
```bash
git init                      # 현재 폴더를 git 저장소로 초기화
git status                    # 현재 상태 확인
git add .                     # 전체 파일 스테이징
git commit -m "커밋 메시지"    # 커밋
```

### 깃허브 연결 & 푸시 (처음 1회)
```bash
git remote add origin https://github.com/유저명/레포명.git
git branch -M main
git push -u origin main
```

### 이후 수정사항 푸시
```bash
git add .
git commit -m "수정 내용"
git push origin main
```

## git status란?
현재 작업 폴더의 상태 확인 명령어

| 상태 | 설명 |
|------|------|
| `Untracked files` | Git이 아직 추적하지 않는 새 파일 |
| `Changes not staged` | 수정됐지만 add 안 한 파일 |
| `Changes to be committed` | add 완료, 커밋 대기 중 |

## SourceTree 사용법

### 로컬 저장소 등록
- SourceTree 실행 → **Add** (기존 폴더) 또는 **Create** (새로 생성)
- 해당 폴더를 드래그하여 넣기

### 스테이징 취소 (git add 취소)
1. **File Status** 탭 클릭
2. **Staged Files**에서 파일 우클릭 → **Unstage**

### 파일 수정 내용 되돌리기
1. **Unstaged Files**에서 파일 우클릭 → **Discard**
> ⚠️ 수정 내용이 완전히 삭제되므로 주의!

### History에서 이전 커밋으로 원복
1. **History** 탭 클릭
2. 되돌아갈 커밋 우클릭 → **Reset to this commit**

| 옵션 | 설명 | 파일 유지 |
|------|------|----------|
| `Soft` | 커밋만 취소 | ✅ 유지 |
| `Mixed` | 커밋 + 스테이징 취소 | ✅ 유지 |
| `Hard` | 커밋 + 파일 모두 취소 | ❌ 삭제 |

> ⚠️ Hard Reset은 복구 불가!

## .gitignore

### 역할
Git이 **무시할 파일/폴더**를 지정 (커밋/푸시 시 제외됨)

### 자주 쓰는 패턴

| 패턴 | 설명 |
|------|------|
| `secrets.yaml` | 특정 파일 무시 |
| `/secrets.yaml` | 최상위 폴더의 해당 파일만 무시 |
| `*.yaml` | 특정 확장자 전체 무시 |
| `!main.yaml` | 예외 처리 (무시하지 않음) |
| `logs` | logs 파일/폴더와 그 내용 |
| `logs/` | logs 폴더와 그 내용 |
| `logs/*.txt` | logs 폴더 안의 txt 파일들 |
| `node_modules/` | npm 패키지 폴더 |
| `.DS_Store` | Mac 시스템 파일 |
| `*.env` | 환경변수 파일 |

### 주의사항
- 이미 커밋된 파일은 .gitignore 추가해도 무시 안 됨
- 이미 올라간 파일 제거 시:
```bash
git rm --cached 파일명
```
