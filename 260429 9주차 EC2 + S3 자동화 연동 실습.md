# AWS CLI 기반 EC2 + S3 자동화 연동 실습

## 수업 목표

- AWS CLI 명령어 사용법 이해
- UserData 스크립트 활용
- EC2 인스턴스 자동화 구성
- EC2 부팅 시 자동 환경 구성

---

## 실습 구성도

```
EC2 시작 → UserData 실행 → EC2 시작 완료
               ↑↓
          Amazon S3
       (index.php 저장)
```

> EC2가 시작될 때 UserData 스크립트가 실행되어 S3에서 `index.php`를 다운로드하고 웹 서버를 자동으로 구성한다.

---

## S3 버킷에 index.php 업로드

아래 파일을 S3 버킷에 미리 업로드해야 한다.

```php
<html>
<head>
  <title>AWS EC2 PHP</title>
  <meta http-equiv=Content-Type content=text/html; charset=ISO-8859-1>
</head>
<body>
  <h1>Welcome to the AWS EC2 PHP</h1>
  <p>
    <?php
      print "The Current Date and Time is: <br/>";
      print date("g:i A l, F j, Y.");
    ?>
  </p>
  <h2>PHP Information</h2>
  <?php phpinfo(); ?>
</body>
</html>
```

## EC2 인스턴스 생성

| 항목 | 값 |
|------|-----|
| 이름 | `sgu-[계정번호]-ec2-s3test` (예: `sgu-202601-ec2-s3test`) |
| OS 이미지 | Amazon Linux 2 |
| 인스턴스 타입 | t3.nano |
| 키페어 | 자신이 생성한 키페어 선택 |
| 보안그룹 | 자신이 생성한 보안그룹 선택 |

### IAM 인스턴스 프로파일 설정

- 고급 세부 정보 클릭
- IAM 인스턴스 프로파일 → 자신의 계정 선택
  - 예: `SafeInstanceProfileForUser-sgu-2025-XX`

### UserData 스크립트 입력

```bash
#!/bin/bash
yum install httpd php -y
aws s3 cp s3://sgu-[계정번호]-s3/index.php /var/www/html --region ap-northeast-2
systemctl restart httpd
```

> ⚠️ `sgu-[계정번호]-s3` 부분은 자신의 S3 버킷 이름으로 변경할 것

스크립트 입력 후 **인스턴스 시작** 버튼 클릭

## AWS CLI란?

**Amazon Web Services Command Line Interface**

- AWS 리소스(S3, EC2, IAM 등)를 터미널 또는 명령 프롬프트로 제어하는 도구
- 콘솔 클릭 없이 명령어만으로 S3 버킷 생성, 파일 업로드, EC2 실행 등의 작업 가능

---

## AWS S3 CLI 주요 명령어

### 목록 조회

```bash
# 전체 버킷 목록 확인
aws s3 ls

# 특정 버킷 안의 파일 목록
aws s3 ls s3://버킷이름/
```

### 파일 업로드 / 다운로드

```bash
# 파일 업로드 (로컬 → S3)
aws s3 cp 로컬경로 s3://버킷이름/경로

# 파일 다운로드 (S3 → 로컬)
aws s3 cp s3://버킷이름/경로 로컬경로
```

### 디렉토리(폴더) 업로드 / 다운로드

```bash
# 폴더 다운로드 (재귀적으로 전체 다운로드)
aws s3 cp s3://my-bucket/folder ./local-folder --recursive

# 폴더 업로드 (재귀적으로 전체 업로드)
aws s3 cp ./local-folder s3://my-bucket/folder --recursive
```
