# S3 정적 호스팅 실습

## 실습 개요

생성된 S3 버킷을 활용하여 정적 웹 호스팅을 구성한다.

- `index.html` 생성
- S3 버킷에 `index.html` 및 이미지 파일 업로드
- 버킷의 퍼블릭 액세스 허용

---

## 1. S3 퍼블릭 액세스 허용

**경로:** 버킷 → 권한 → 편집

- "모든 퍼블릭 액세스 차단" 체크 해제
- 변경 사항 저장

---

## 2. index.html 생성

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <title>S3 정적 웹사이트 실습</title>
</head>
<body>
  <h1>Amazon S3 정적 웹사이트 호스팅 실습</h1>
  <p>이 페이지는 S3 버킷에서 호스팅되고 있습니다.</p>
  <h2>업로드한 이미지:</h2>
  <img src="class/img/image.png" alt="S3에 업로드한 이미지">
</body>
</html>
```

---

## 3. 파일 업로드

| 파일 | 업로드 위치 |
|------|------------|
| `index.html` | 버킷 루트 (`/`) |
| `image.png` | `class/img/` 폴더 |

> S3 콘솔에서 **폴더 만들기** 버튼으로 `class/` → `img/` 순서로 폴더 생성 후 업로드

---

## 4. 객체 URL로 접근 시 AccessDenied 발생

`index.html`의 **객체 URL**(`https://버킷명.s3.ap-northeast-2.amazonaws.com/index.html`)로 접근하면 아래 오류 발생:

```xml
<Error>
  <Code>AccessDenied</Code>
  <Message>Access Denied</Message>
</Error>
```

**이유:** 퍼블릭 액세스를 허용해도, **버킷 정책(Bucket Policy)** 이 없으면 객체는 기본적으로 비공개 상태

---

## 5. S3 버킷 정책(Bucket Policy) 개요

- S3 리소스(버킷 및 객체)에 대한 **접근 제어 정책**
- **JSON 형식**의 문서
- 버킷 단위로 설정 → 버킷 내 모든 객체에 적용 가능
- IAM 정책과의 차이:

| 구분 | 설명 |
|------|------|
| **IAM 정책** (Identity-based) | 사용자/그룹/역할에 부여 → "누가 무엇을 할 수 있는가" |
| **버킷 정책** (Resource-based) | 리소스에 직접 부여 → "이 리소스에 누가 접근 가능한가" |

---

## 6. 버킷 정책 추가

**경로:** 버킷 → 권한 → 버킷 정책 → 편집

### 버킷 전체 퍼블릭 허용

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicWebsiteAllow",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::sgu-202500-s3/*"
    }
  ]
}
```

---

## 7. 응용 정책 예시

### 특정 경로만 허용

`index.html`과 `class/img/` 폴더만 공개하고 싶을 때:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicWebsiteAllow",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": [
        "arn:aws:s3:::sgu-202500-s3-test/index.html",
        "arn:aws:s3:::sgu-202500-s3-test/class/img/*"
      ]
    }
  ]
}
```

---

### 특정 IP만 허용

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicWebsiteAllow",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": [
        "arn:aws:s3:::sgu-202500-s3-test/index.html",
        "arn:aws:s3:::sgu-202500-s3-test/class/img/*"
      ],
      "Condition": {
        "IpAddress": {
          "aws:SourceIp": "123.123.123.123"
        }
      }
    }
  ]
}
```

---

### 특정 폴더 다운로드 제한

`class/test/` 폴더는 업로드는 허용하되 다운로드(GetObject)는 차단하고 싶을 때:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicWebsiteAllow",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": [
        "arn:aws:s3:::sgu-202500-s3-test/index.html",
        "arn:aws:s3:::sgu-202500-s3-test/class/img/*"
      ],
      "Condition": {
        "IpAddress": {
          "aws:SourceIp": "123.123.123.123"
        }
      }
    },
    {
      "Sid": "DenyGetObject",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::sgu-202500-s3-test/class/test/*"
    }
  ]
}
```

> `"Effect": "Deny"` + `"Action": "s3:GetObject"` 조합으로 특정 경로의 읽기를 명시적으로 차단
